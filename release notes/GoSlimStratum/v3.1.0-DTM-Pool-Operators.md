# Direct-to-Miner (DTM) Mode — Best Practices Guide

## What is DTM Mode?

Block rewards go directly to the miner's wallet via the coinbase transaction. No payout system needed — miners get paid the moment a block is found (spendable after maturity confirmations). This is fundamentally different from pool mode where the pool collects rewards and distributes payments.

---
## For Pool Operators (Running a Pool for Others)

You're running GoSlimStratum as a pool where other miners connect and you collect a fee. Example: Doug running a public pool with an Enterprise license.

### Enabling DTM

1. Navigate to each coin's **Configuration** page
2. Toggle DTM on (requires a license for full DTM, or accept revenue share on built-in coins)
3. Save and reload the coin pool
4. Repeat for each coin pool you operate

### Recommended Configuration

| Setting | Value | Why |
|---------|-------|-----|
| `pool_fee_percent` | **1-2%** | Your revenue. Collected as a second coinbase output automatically. |
| `max_job_history` | **5** | Per-miner job history. Lower saves memory, especially with many miners. |
| `payout.enabled` | **true** | Enables maturity tracking and earnings recording for your dashboard. |
| `payout.check_interval_seconds` | **120** | Faster maturity status updates. Miners like seeing Immature → Spendable quickly. |
| `mining.address` | **Your fee collection wallet** | This is where your pool fee goes. Does NOT need to be the node wallet — use any wallet you control for easy management. |

### Pool Fee Address

In DTM mode, the `mining.address` is your **fee collection address**, not the payout source. It does NOT need to belong to the node wallet because:
- There are no payout transactions to sign — rewards go directly to miners via coinbase
- Your fee is a separate coinbase output sent to this address automatically
- Use an external wallet (Exodus, hardware wallet, exchange) for easy fee management

**Note**: In pool mode this would trigger a "Wallet Address Mismatch" warning banner. In DTM mode, the warning is intentionally suppressed because external addresses are expected.

### What Your Miners Will See

- **Dashboard**: Blocks found with confirmation countdown, then "Spendable" badge
- **Earnings page**: "Direct Payouts" table showing their blocks, rewards, and status
- **Their wallets**: Block reward minus your pool fee, appearing immediately as unconfirmed

### Coinbase Output Structure

Each block your pool finds has these outputs:
```
Output 1: Miner reward (block reward - pool fee - revenue share)  → miner's wallet
Output 2: Pool fee (if > 0%)                                      → your fee address
Output 3: Revenue share (if no license, 0.5%)                     → embedded GSS address
Output 4: Witness commitment (SegWit coins)                        → zero-value
```

With a license and 2% fee:
```
Output 1: 98% of block reward  → miner's wallet
Output 2: 2% of block reward   → your fee address
Output 3: Witness commitment    → zero-value (SegWit)
```

## Configuration Reference

### Per-Coin Config (config.json)

```json
{
  "COIN": {
    "enabled": true,
    "enable_dtm": true,
    "mining": {
      "address": "YOUR_FEE_OR_NODE_ADDRESS",
      "max_job_history": 5
    },
    "payout": {
      "enabled": true,
      "pool_fee_percent": 1.0,
      "check_interval_seconds": 120,
      "maturity_confirmations": 100
    }
  }
}
```

### Key Fields

| Field | Description |
|-------|-------------|
| `enable_dtm` | Enable Direct-to-Miner mode for this coin |
| `mining.address` | Pool fee collection address (solo: node wallet, operator: any wallet) |
| `mining.max_job_history` | Per-miner job history depth (default 20, recommended 5 for DTM) |
| `payout.enabled` | Enable maturity tracking and earnings dashboard |
| `payout.pool_fee_percent` | Fee percentage (solo: 0%, operator: 1-2%) |
| `payout.check_interval_seconds` | How often to check block maturity (default 300, recommended 120 for DTM) |
| `payout.maturity_confirmations` | Confirmations before block is "Spendable" (coin-dependent) |
